# TCP vs UDP


|      | TCP (Transmission Control Protocol) | UDP (User Datagram Protocol) |
| ---- | ----------------------------------- | ---------------------------- |
| 连接方式 | 面向连接，需要三次握手                         | 无连接，直接发送                     |
| 可靠性  | 可靠，丢包会重传                            | 不可靠，丢包不重传                    |
| 顺序保证 | 保证数据按顺序到达                           | 不保证顺序                        |
| 数据模型 | 字节流 Stream                          | 数据报 Datagram                 |
| 消息边界 | 不保留消息边界                             | 保留消息边界                       |
| 流量控制 | 有                                   | 没有                           |
| 拥塞控制 | 有                                   | 没有                           |
| 开销   | 较大                                  | 较小                           |
| 延迟   | 相对较高                                | 相对较低                         |
| 速度   | 相对慢，但稳定                             | 快，但可能丢包                      |
| 典型问题 | Head-of-line blocking               | 丢包、乱序、重复包                    |
| 适合场景 | 数据必须完整、正确、有序                        | 低延迟、允许少量丢包                   |


---

# 流量控制 Flow Control

发送方不能发太快，否则接收方的 buffer 可能无法处理。

TCP 里接收方会告诉发送方：我现在还能接收多少数据 (receive window / rwnd)。


---

# 拥塞控制 Congestion Control

即使接收方很强，网络中间的 router、switch、链路也可能承受不了太多流量，导致丢包、延迟上升。

TCP 会根据网络情况动态调整发送速度 (congestion window / cwnd)。

> 如果网络看起来很顺畅，就慢慢加大发送量；如果出现丢包、超时，就认为网络拥塞，降低发送量。
