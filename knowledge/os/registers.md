# 什么是寄存器

寄存器（register）是 CPU 内部的一小块、速度极快的存储单元。它们的**容量通常很小，但访问速度远快于内存**。

在操作系统语境下，寄存器特别重要，因为：

- CPU 正在执行哪条指令，要靠寄存器记录
- 当前函数调用的栈位置，要靠寄存器记录
- 计算过程中的中间结果，常常先放在寄存器里
- 线程/进程发生上下文切换时，必须保存和恢复寄存器状态

---

# 为什么 OS 需要关心寄存器

操作系统要管理多个线程和进程，而 CPU 在某一时刻只能执行一个线程（单核情况下）。

当发生以下情况时，OS 需要切换执行对象：

- 时间片用完
- 线程阻塞
- 中断到来
- 系统调用进入内核
- 调度器决定切换到别的线程

这时，CPU 不能“忘记”当前线程运行到哪里了，所以需要保存执行现场。这个“执行现场”里最核心的内容就是各种寄存器的值。

因此，寄存器状态是上下文切换（context switch）的核心组成部分之一。

---

# 寄存器分类

常见可以分成以下几类：

1. 控制执行流的寄存器
   - PC（Program Counter）
   - SP（Stack Pointer）

2. 保存计算数据的寄存器
   - 通用寄存器（General-Purpose Registers）

3. 保存 CPU 状态的寄存器
   - 状态寄存器 / 标志寄存器（Status / Flags Register）

4. 和内存管理相关的寄存器
   - 页表基址寄存器等

5. 浮点和向量计算相关寄存器
   - FPU / SIMD / Vector Registers

---

# Program Counter

PC 用来保存下一条将要执行的指令地址，它告诉 CPU 接下来应该去哪一块内存取指令。

线程暂停后还能继续运行，最关键的一点就是必须知道它上次执行到哪了，而这个“位置”就是 PC 记录的内容。如果没有 PC，线程切换回来时就不知道从哪条指令继续执行。

PC 不只是简单加一，它会随着控制流变化而变化：

- 顺序执行：PC 指向下一条指令
- `if` / `while` / `for`：跳转到不同代码位置
- 函数调用：跳到函数入口
- 函数返回：恢复到调用点后的位置
- 中断 / 异常：跳到中断或异常处理入口
- 系统调用：切入内核处理逻辑

所以 PC 本质上决定了程序控制流。

---

# Stack Pointer

SP 保存当前栈顶的位置。线程通常各自拥有独立的栈，因此 SP 也是线程执行上下文里非常核心的一部分。

栈（stack）通常用于保存：

- 函数返回地址
- 局部变量
- 部分函数参数
- 保存的寄存器
- 调用链上的栈帧（stack frame）

当函数被调用时，通常会在栈上创建一个新的栈帧。线程切换回来时，不只是要知道“执行到哪”，还要知道：

- 当前函数调用层级是什么
- 局部变量放在哪里
- 返回地址在哪里
- 栈帧应该从哪里恢复

所以 SP 决定“从哪个栈状态继续”。

---

# 通用寄存器（General-Purpose Registers）

通用寄存器用来存放临时数据和计算中间结果。  
- 加法、减法、中间变量
- 函数参数
- 函数返回值
- 内存地址
- 循环计数

线程运行时，很多中间值都在这些寄存器里。如果线程被切走，而这些值没有保存，那么下次回来时计算就错了。所以在上下文切换时，OS 通常要保存必要的通用寄存器，并在恢复线程时重新装回去。

---

# 状态寄存器 / 标志寄存器（Status / Flags Register）

状态寄存器保存 CPU 当前的一些状态信息。  

- 零标志（Zero Flag）
- 进位标志（Carry Flag）
- 符号标志（Sign Flag）
- 溢出标志（Overflow Flag）
- 中断允许位
- 特权相关状态位

这些标志位也是线程执行现场的一部分。如果不保存，线程恢复后，条件判断、算术状态、中断状态等都可能混乱。

---

# Base Pointer / Frame Pointer

很多架构或编译器会使用一个寄存器来指向当前栈帧的基准位置。一个函数的栈帧里通常有：

- 返回地址
- 保存的旧帧指针
- 局部变量
- 临时空间

使用 BP/FP 作为固定参考点，可以更方便地访问局部变量和参数。例如可以通过类似“相对 FP 的偏移”来访问变量。

---

# Memory Management Registers

现代 OS 使用虚拟内存。进程访问的是虚拟地址，而不是直接访问物理地址。CPU 需要某种方式知道当前应该使用哪套页表来做地址翻译。

页表本身一般属于进程地址空间的一部分，因此更偏“进程资源”，通常和 PCB/进程内存描述结构相关。但真正参与 CPU 地址翻译的，是硬件寄存器里的当前页表基址信息。

---

# 特权级相关寄存器

CPU 通常支持不同特权级，比如：

- 用户态（user mode）
- 内核态（kernel mode）

某些寄存器或状态位会反映当前 CPU 所处的特权级，或者在模式切换时参与工作。
